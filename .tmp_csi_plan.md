# CSI Target Reformulation Plan

## Background
- HRRR diagnosis: GHI MSE_Skill=0.82 vs persistence, but corr(HRRR_error, PV_error)=0.02
- Current best model (E1): t+1h day_nRMSE=0.261, t+24h day_nRMSE=0.335
- Problem: PV model doesn't effectively use HRRR GHI forecast

## Hypothesis
Changing prediction target from `pv_power_kw` to `k = pv_power / clear_sky_power` (CSI) will:
1. Remove deterministic diurnal cycle, letting model focus on weather attenuation
2. Allow HRRR GHI to enter via `hrrr_csi_valid = HRRR_GHI_valid / clear_sky_GHI_valid`
3. Improve Mixed/Overcast performance

## Experiment Matrix (5 groups, all on t+24h)

| ID | Target | HRRR Usage | Purpose |
|----|--------|-----------|---------|
| C0 | PV power | None (E1 baseline) | Control |
| C1 | CSI = PV / clear_sky_power | None | Isolate target reformulation effect |
| C2 | CSI target | hrrr_csi_valid as feature | Test HRRR skill transfer to PV |
| C3 | Physical baseline | PV_pred = clear_sky_power × hrrr_csi_valid | Direct HRRR scaling (no ML) |
| C4 | CSI target | hrrr_csi_valid + lag/ramp/history | Full candidate |

## clear_sky_power Computation
- Plan A: pvlib chain (clearsky irradiance → POA transposition → temperature → capacity)
- Plan B: Training-set empirical P95 envelope (fallback if params incomplete)
- Site: Golden CO, lat=39.74, lon=-105.18, alt=1730m, capacity=1.12kW
- Note: Need to check if tilt/azimuth params are available in config

## CSI Target Protection
- Filter: solar_elevation > 5 AND clear_sky_power > 0.05 × capacity
- Clip k to [0, 1.2] (allow cloud enhancement margin)
- Post-process: clip k_pred to [0, 1.2] before restoring PV
- Report: % filtered, % clipped, % k>1.0

## HRRR Availability Check
- hrrr_valid_time == pv_valid_time
- hrrr_issue_time <= pv_origin_time
- Focus on t+24h since HRRR lead_time is 24-44h

## Evaluation
- Restore to PV before scoring: PV_pred = k_pred × clear_sky_power_valid
- Metrics: RMSE/capacity, daily energy MAE, Mixed scenario RMSE
- Weather stratification: Clear/Mixed/Overcast
- Check PV error vs HRRR error correlation (baseline: corr=0.02 from diagnosis)
- Update inspection_predictions.parquet with new experiments
