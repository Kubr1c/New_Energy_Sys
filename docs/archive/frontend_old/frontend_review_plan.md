## 核心问题诊断

### 问题一：导航命名模糊，功能边界不清

> [!WARNING]
> 侧栏 6 个导航项的命名和功能分界让人难以快速理解每个页面的核心用途。

| 当前导航 | 实际内容 | 问题 |
|---------|---------|------|
| 预测监控 Overview | KPI卡片 + 预测曲线 + 站点信息 | "监控" 暗示实时性，但实际是静态历史数据展示 |
| 模型评估 Models | 排行榜表格 + Bar图 + Radar图 | 功能明确，但缺乏可操作性 |
| 调度收益 Dispatch | Rawhide实站 + S13原型治理 + Pareto表 + 退化回放 | **严重过载**：一个页面塞了 4 个完全不同的功能模块 |
| 配置治理 Governance | Pareto散点 + 热力图 + 配置表格 | 和 Dispatch 页面内容高度重叠（都有 Pareto 表） |
| 数据运维 Data | 数据质量 + 特征重要性 + 任务触发 | "运维" 这个词太技术化，3 个功能缺乏逻辑关联 |
| 报告归档 Reports | Markdown 报告浏览器 | 功能明确但缺乏与其他模块的关联 |

### 问题二：Dispatch 页面严重信息过载

> [!CAUTION]
> [DispatchSimulation.vue](file:///c:/Project/New_Energy_Sys/frontend/src/views/DispatchSimulation.vue) 有 **327 行模板代码**，包含：
> - Rawhide 实站英雄区（站点信息 + 事实面板）
> - 边界说明横幅
> - 6 个 KPI 指标卡
> - 增量收益图表
> - 退化回放面板（含 SOH、循环、容量衰减等子指标）
> - Pareto 配置表格
> - Stage13 原型策略卡片（4 张）
> - 策略评分对比图表
> - 治理雷达图
> 
> 单个页面承载了约 **15+ 个独立信息模块**，用户进入后完全不知道应该先看什么。

---

### 问题三：缺乏用户引导和操作流程

- **没有"做什么"的引导**：用户进入任何页面，只看到大量数据，但不知道应该关注什么、采取什么行动。
- **没有结论/洞察摘要**：每个页面只是把数据摆出来，缺乏"所以呢？"——例如 "模型 A 比模型 B 好 23%"、"推荐配置 X 可节省 €200"。
- **交互操作几乎为零**：除了 Overview 的天数选择器和 Models 的特征集筛选器，用户没有任何可操作的控件。

---

### 问题四：中英文混杂不一致

- 导航：`预测监控 Overview`、`模型评估 Models` —— 中英同时出现
- 页面内容：标题用中英混合 `"策略评分对比 — Strategy Scores"`，但正文全是技术术语
- 数据标签：`nRMSE`、`MAE (kW)` 全英文，但说明文字 `"主模型测试集 nRMSE"` 是中文
- **建议**：统一使用中文为主（面向国内学术演示），英文仅用于专业术语

---

### 问题五：视觉层次扁平，缺乏焦点

- 所有 `glass-card` 外观一致，没有主次之分
- KPI 数值没有大小/颜色对比来突出关键指标
- 图表标题和说明文字字号接近（14px vs 12px），层次感弱
- 缺少趋势箭头、变化百分比等"一眼可读"的视觉元素

---

### 问题六：功能缺失

- **没有数据导出**：用户无法下载图表或表格数据
- **没有时间范围切换**：Overview 只能切 7/14/30 天，不能自定义
- **没有对比模式**：Models 页只能看排行，不能选择 2 个模型做 A/B 对比
- **没有搜索/筛选**：Reports 页没有搜索框，阶段多了找不到
- **缺少帮助提示**：技术指标（nRMSE、MAE、Pareto前沿）没有 tooltip 解释

## 改进方案

### 方案一：导航重构 —— 从 6 个精简到 4 个核心模块

```
当前 (6个)                    改进后 (4个)
─────────────────────       ─────────────────────
预测监控 Overview     ──→    📊 系统总览
模型评估 Models       ──→    🤖 模型对比
调度收益 Dispatch     ──┐    ⚡ 调度分析 (合并+拆分Tab)
配置治理 Governance   ──┘    
数据运维 Data         ──→    📋 数据与报告 (合并Data+Reports)
报告归档 Reports      ──┘    
```

**具体调整**：

| 新模块 | 内容 | Tab/子页面 |
|--------|------|-----------|
| **系统总览** | 保持现有 Overview，但增加"关键结论"摘要区 | 单页，顶部增加"一句话结论" |
| **模型对比** | 保持排行榜 + 新增 A/B 对比模式 | `排行榜` / `详细对比` 两个 Tab |
| **调度分析** | 合并 Dispatch + Governance | `策略评估` / `配置优化` / `退化分析` 三个 Tab |
| **数据与报告** | 合并 Data + Reports | `数据质量` / `实验报告` / `任务管理` 三个 Tab |

---

### 方案二：每个页面增加"核心结论"区

在每个页面顶部增加一个醒目的结论横幅：

```
┌──────────────────────────────────────────────────┐
│ 💡 核心结论                                        │
│ LightGBM tuned 模型在测试集上 nRMSE 最优（0.0342），│
│ 较基线 LinearRegression 提升 67.2%。               │
│ 推荐在 full_features 特征集下部署。                  │
└──────────────────────────────────────────────────┘
```

---

### 方案三：增强交互能力

| 页面 | 新增交互 |
|------|---------|
| 系统总览 | 日期范围选择器（DatePicker），点击 KPI 卡片可跳转到详情页 |
| 模型对比 | 勾选 2-3 个模型进行逐指标对比，对比结果用表格+差值高亮 |
| 调度分析 | Tab 切换不同分析维度，策略卡片可展开/折叠 |
| 数据与报告 | 报告搜索框，数据表格支持列排序和筛选 |

---

### 方案四：统一语言风格

> [!IMPORTANT]
> 建议统一为**中文优先**风格（适合毕设答辩/学术演示）：
> - 导航只用中文：`系统总览`、`模型对比`、`调度分析`、`数据报告`
> - 专业术语保留英文缩写并附 tooltip：`nRMSE (?)`
> - 图表标题纯中文：`"功率预测曲线"` 而非 `"PV 功率预测 vs 实际 — Prediction vs Actual"`
> - 按钮文本纯中文：`"登录"` 而非 `"登录 Sign In"`

---

### 方案五：视觉优化

1. **KPI 卡片增加趋势指示**：用 `↑ 23%` `↓ 5%` 显示相对变化
2. **关键数值放大**：最重要的 1-2 个 KPI 使用 36px+ 字号，其他用 22px
3. **颜色语义化**：好=绿色，差=红色，中性=蓝色，统一应用
4. **添加分隔线/标题**：在长页面中增加明确的功能分区标题
5. **空状态优化**：增加引导性插图，而非纯文字
