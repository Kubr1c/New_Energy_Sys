# 前端问题修复计划

## 目标

修复演示视频中暴露的前端核心问题，使系统具备真实可用的导航、稳定加载体验、可读的数据图表、清晰报告入口和更适合验收演示的交互路径。

## 实施优先级

1. P0：修复登录态响应式与导航显示问题。
2. P0：修复路由切换时的空白/黑屏体验。
3. P1：优化 Dashboard、Model、Data、Report 页面的可读性和信息结构。
4. P2：提高暗色视觉对比度、补充业务解释、优化演示录屏路径。

## 关键改动

### 登录态与主框架

- 新增轻量响应式认证状态模块 `src/stores/authState.js`。
- 登录成功后同时写入 `localStorage` 和响应式状态。
- 登出、刷新恢复、401 拦截统一走认证状态模块。
- `App.vue` 基于响应式登录态显示侧边栏、顶部栏和用户信息。
- 修复导航、页面标题、登录页等主要 mojibake 乱码文案。

### 加载态与页面结构

- 统一 `PageState` 为更亮、更稳定的占位面板。
- 页面加载、空数据、错误状态均显示明确说明和重试入口。
- 减少路由切换时的近黑屏感。

### 核心页面

- Overview：调整为 KPI、主预测图、站点信息、模型状态和紧凑 Pipeline 的首屏结构。
- Model：排行榜保留全量表格，图表聚焦 Top 12 / Top 5，提高坐标与图例可读性。
- Dispatch：补充 Rawhide 边界说明、负收益和退化成本解释，修复货币与中文乱码。
- Governance：补充 Pareto 和热力图解释，修复表格与图表文案。
- Data：按真实后端字段读取质量报告，无字段时显示“字段缺失”，不伪造数据。
- Report：进入页面默认打开第一个可用报告，左侧阶段名支持两行展示。

### 演示录屏

- 录屏用例固定 1920x1080。
- 演示路径改为登录后点击可见侧边栏导航，而不是直接改 URL。
- 输出 WebM 中间文件，再用 `ffmpeg` 转为 MP4。

## 验收标准

- 登录后无需刷新即可看到侧边栏、顶部栏和用户信息。
- 所有核心页面均可通过可见导航访问。
- 页面加载时不出现长时间空白主体。
- 1080p 下主要图表、标题、指标可读。
- Report 页面默认展示一个报告或明确错误状态。
- Data 页面指标缺失时给出原因，不显示无解释的横杠。
- 演示视频为 1920x1080 MP4。

## 测试计划

- `npm run check`
- `npm run build`
- `npm run test:e2e`
- 重新生成 `reports/frontend-demo-1080p.mp4`
- 使用 `ffprobe` 校验 MP4 分辨率为 1920x1080

## 潜在坑点

- 认证状态不能只修登录页；401、登出和刷新恢复必须同步处理。
- Data 页面不允许伪造质量指标，后端字段缺失必须暴露为明确状态。
- 图表减少默认展示数量后，需要保留表格或明细入口，避免丢失科研结果完整性。
- MP4 依赖本机 `ffmpeg`，Playwright 原生视频仍是 WebM。
